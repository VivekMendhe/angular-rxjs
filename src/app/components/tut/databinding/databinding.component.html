1. Display Dynamic Data

This shows how to bind and display dynamic data in the template using Angular interpolation.

{{ employee }}

{{ emp.name }}

{{ emp.position }}

2. String Concatenation

You can concatenate strings with dynamic values using the `+` operator within interpolation.

{{ "My name is: " + emp.name }}

3. Calculations

You can perform calculations within interpolation, like multiplication, division, and more.

{{ "Multiplication: " + 5 * 2 }}

{{ "Addition: " + (5 + 2) }}

{{ "Division: " + 5 / 2 }}

{{ "Subtraction: " + (5 - 2) }}

{{ "Total: " + price * quantity }}

4. Built-in JS Methods

You can use JavaScript's built-in methods, like `slice()`, `length`, and more, inside Angular templates.

{{ "Length of employee: " + employee.length }}

{{ "Slice of employee: " + employee.slice(2, 3) }}

  • toUpperCase(): Converts a string to uppercase.
  • toLowerCase(): Converts a string to lowercase.
  • length: Returns the length of a string.
  • slice(): Extracts a portion of a string.
  • indexOf(): Finds the index of a substring.
  • includes(): Checks if a string contains a specific substring.

5. Display Method Result

You can call methods inside the template and display their return values.

{{ "Total Calculation: " + calculateTotal() }}

{{ "Tax Calculation: " + calculateTax().toFixed(2) + "%" }}

6. Ternary Conditional Operator

Ternary operators allow you to conditionally display values in the template.

7. Where Not to Use Interpolation

Avoid using interpolation in attributes, as it may lead to errors. Interpolation is not supported in attribute bindings and certain properties like `href`, `src`, or `style`. Instead, use property binding for such cases.

Incorrect Usage:

<a href="{{ 'link' }}">Click here</a>

The example above would throw an error because Angular doesn't support interpolation in attributes such as `href` or `src`.

Correct Usage:

<a [href]="link">Click here</a>

In this example, property binding (`[href]`) is used to bind dynamic values to attributes, which is the correct approach.
